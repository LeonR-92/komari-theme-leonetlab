# 更新日志

## 1.2.6 — 2026-07-24

- 修复首访交接的共享朝向死代码：`sharedIntroOrientation` 原声明在 `<script setup>` 顶层，每个组件实例各持一份，dashboard 地球仪永远读不到 intro 朝向（`valid` 恒 false），交接末尾瞬跳回默认角度；现移入模块作用域，dashboard 在交接完成瞬间继承 intro 最后一帧 phi/theta（断言误差 < 0.01 rad）并继续自转。
- 首访交接飞行期间监听窗口 resize，按 rAF 节流重新测量 FLIP 目标矩形，终点跟随最新布局。
- 减少动态效果偏好补全：地球仪停止自动旋转（保留拖拽），卡片/列表 ping 波纹、地球与地图状态脉冲点、页头状态轮换、访客骨架脉冲等无限动画全部静止。
- Ping 图表新增 `-32603` 且消息含 "metric store" 的回退白名单，官方指标存储未初始化时安全回退到 `common:getRecords`，其它内部错误继续上抛。
- 兼容声明扩展到官方 `1.2.6` 与 `1.3.x` 稳定版；Service Worker 缓存升级为 1.2.6，首访会话标记同步升级，老用户更新后重看一次新 intro。
- 浏览器回归新增交接 phi/theta 连续性、画布中心/尺寸连续性和 metric-store 回退断言。

## 1.1.9 — 2026-07-21

- 修复后台默认明暗模式被自动创建的 `appearance` 缓存错误覆盖的问题；只有访客实际点击页头切换后才写入个人覆盖标记。
- 将 Service Worker 缓存升级为 1.1.9，注册时绕过 HTTP 缓存并主动检查更新；新主题资源接管页面后执行一次带会话防循环的刷新。
- 重做访客身份卡收束阶段，先隐藏详细内容，再平滑缩短容器，最后显示紧凑摘要；浏览器测试验证收束帧无长任务与高度突跳。
- 手机端 Ping 探针列表改为自适应网格，390 px 下电信、联通、移动三条线路同屏可点击，不再依赖横向滚动。
- 扩充 Komari 1.2.5-fix1/fix2/1.2.7 浏览器夹具，覆盖后台主题模式、用户覆盖、三探针弹窗、实时 Ping 数据和缓存更新代码路径。
- Chrome DevTools 生产构建审计补齐节点搜索与汇率币种控件的名称标识，消除浏览器表单识别告警。
